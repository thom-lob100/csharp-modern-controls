## 목적

이 저장소의 기존 WinForms 화면(폼)을 Modern.Lab 모던 컨트롤로 변환한다.
**화면의 겉모습(컨트롤)만 교체하고, 업무 동작은 변환 전과 완전히 동일해야 한다.**

## 절대 규칙

1. **한 번에 폼 하나만** 변환한다. 지시받지 않은 폼/파일은 절대 수정하지 않는다.
2. **업무 코드는 수정 금지.** 서버 요청/응답(전문, 미들웨어, DB 호출), 데이터 가공,
   계산, 검증 로직은 단 한 줄도 바꾸지 않는다. 컨트롤 교체 때문에 꼭 필요한
   최소 보정(미지원 멤버의 대체)만 허용되며, 그때도 절차서의 대체 표를 따른다.
3. **레이아웃 구조 유지.** `Panel`/`TableLayoutPanel`/`SplitContainer` 같은 배치
   컨테이너의 구조·Dock·Anchor는 그대로 둔다. 모던 컨트롤은 잎(leaf) 위젯 교체다.
4. **색·폰트·크기 하드코딩 금지.** 모던 컨트롤의 외형은 라이브러리 테마(토큰)가
   결정한다. `BackColor`/`ForeColor`/`Font`를 새로 지정하지 않으며, 교체 과정에서
   기존 지정이 있었다면 제거한다(절차서 3-3단계).
5. **빌드가 깨진 상태로 작업을 끝내지 않는다.** 매 폼 변환 후 위 빌드 명령으로
   확인하고, 오류가 있으면 해결한 뒤 보고한다.
6. 기존 코드의 스타일(들여쓰기, 네이밍, 주석 언어)을 그대로 따른다.
7. 확신이 없거나 절차서/마이그레이션 문서에 대체 방법이 없는 멤버를 만나면
   임의로 구현하지 말고 **"보류 항목"으로 보고**한다.
8. **C# UI 소스 폴더 밖은 수정 금지.** 특히 Java 서버 소스 폴더
   (`{{수정 금지 폴더 — 예: C:\smartAOS\biz}}`)는 이해를 위해 읽는 것까지만
   허용하며, 그 안의 어떤 파일도 만들거나 고치지 않는다.
9. **작업 전 백업.** 폼 변환을 시작하기 전에 수정 예정 파일을
   `_backup\<날짜>_<폼이름>\` 폴더로 복사한다. (git을 쓸 수 있는 환경이라면
   브랜치 생성 + 커밋으로 대체해도 된다.)

## 작업 절차 (요약 — 상세는 절차서)

1. **조사**: 대상 폼의 컨트롤 목록을 만들고 매핑 표(변환 계획)를 작성해 보고한다. 사람이 승인하기 전에는 코드를 수정하지 않는다.
2. **변환**: `.Designer.cs`의 타입 교체 → 코드비하인드 미지원 멤버 보정 → 빌드.
3. **보고**: 변경 파일 목록, 바꾼 컨트롤 표, 보류 항목, 사람이 실행해서 확인할 체크리스트를 출력한다.
